# Architecture

## Layers
1. Vision
2. Canon
3. Principles
4. Data
5. Prompts
6. Workflows
7. Tools
8. Outputs

## System model
- Documentation defines the rules.
- Structured data stores the approved knowledge.
- Prompts compile the knowledge into creative instructions.
- Workflows define the production route.
- Tools evaluate, compile, and recommend.
- Outputs are the generated pieces, exports, and assets.

## Design intent
The system must support internal use, experimentation, production, and future automation without losing the Illo & Killo canon.

## Current priority
Keep the canon stable while increasing structure and reducing ambiguity.
