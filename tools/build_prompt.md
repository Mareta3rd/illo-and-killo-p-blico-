# Build prompt

## Objective
Assemble the final production prompt from brief, structured data and workflow context.

## Inputs
- brief
- character
- gag
- route
- objects
- fauna
- heritage
- palette
- support
- destination
- constraints

## Steps
1. Load the validated brief.
2. Load the selected canonical data.
3. Apply route-specific rules.
4. Apply support-specific rules.
5. Apply destination-specific rules.
6. Merge all approved elements into a final prompt.
7. Preserve canon and eliminate ambiguity.

## Rule
The builder may adapt wording for clarity, but it must not invent new canon or weaken existing constraints.
